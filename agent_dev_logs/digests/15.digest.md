# Issue #15 — Digest (post eval-feature integration)

## Goal
Retrain the 5 best models on the **new dataset (N−318 pages)** and confirm the new release as
canonical: each refinetuned v*.4 model must reproduce its original accuracy on the **same held-out
test pages (minus 318)**.

## Core mechanism
Reproduce the *exact* original train/dev/test split (impossible with the stochastic
`split_data_80_10_10`). The split is read from `licensed_crossval_folds_CUT.csv` — `PNG` column =
exact filename, `fold1..fold5` cells = `train`/`dev`/`test`, with the 318 pages already removed.
Pages absent from the CSV are dropped automatically (that is how the −318 is applied).

## Usage
```bash
# retrain one model on its exact original fold (minus 318)
python3 run.py --train -rev v4.4 --folds_csv path/to/licensed_crossval_folds_CUT.csv
# evaluate the same model on ONLY that fold's held-out test pages
python3 run.py --eval  -rev v4.4 --folds_csv path/to/licensed_crossval_folds_CUT.csv
```
`--eval --folds_csv` is single-model only (`-rev` / `--fold_column`), **not** `--best` — each best
model uses a different fold, so an ensemble has no shared held-out test set.

## Notable finding (bug fixed)
Under the pinned `pandas>=3.0.3`, `Series.astype(str)` **preserves NA** (it is not the literal
`"nan"`). The licensed CSV has ~225 blank `fold1` cells, so the merged `split_data_from_folds`
crashed on `.strip()` — breaking **both train and eval** on the real data. Now guarded with
`pd.isna` + `str(...)`.
